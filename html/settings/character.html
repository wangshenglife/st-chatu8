支持的输入格式（点击展开）
  • 旧格式角色标记：$角色名字-sfw-upperbody$、$角色名字-nsfw-upperbody-sfw-lowerbody$
  • JSON 角色标记：${"name":"角色名字","angle":"from front","upperBody":"sfw","lowerBody":"hidden"}$
  • 多角色（一段 tags 里多个 $...$）：$小红-sfw-upperbody$, standing, $小蓝-sfw-upperbody$, smiling
  • 调试用 · 仅识别名字：角色名字、角色名字[当前]（实际替换流程不认这种裸名字）
测试只做“名字匹配”：从输入中提取角色名字，再按启用列表检测是否命中，不会执行完整的替换流程。
正式解析要求 $...$ 内必须是「名字 + 旧格式后缀」或 JSON 对象；$角色名字$ 这种裸名字在实际替换时会被原样保留。
若输入里存在多段 $...$，会自动按多角色方式逐段识别，并以表格形式展示每段结果。
{{角色启用列表}} 触发规则说明 & 与「角色触发」的区别（点击展开）
  • 触发来源：仅扫描当前「角色启用预设」里登记的角色，不会走通用列表，不会走全局预设兜底。
  • 匹配方式：对每个启用角色的中文名 / 英文名做子串包含。
  • 名字分割：名字支持用 | 分割多个别名，任意一个命中即视为触发。
  • 大小写 / 空格：均已忽略（触发文本和名字都会去掉空白并转小写后比较）。
  • 结果数量：一次可命中多个角色；所有命中角色都会被包含进 {{角色启用列表}} 文本。
与「角色触发（$...$ 替换）」的差别：
  • 列表是否必须启用：列表触发 必须在启用列表里；角色触发启用列表只决定优先级，兜底会匹配所有预设。
  • 匹配算法：列表触发只做字符串包含；角色触发使用 normalizeName + 相似度评分。
  • 命中数量：列表触发可以命中多个；角色触发每个 $...$ 片段只取一个最佳匹配。
  • 输入形态：列表触发接受任意自然文本；角色触发要求 $...$ 内是旧格式后缀或 JSON 对象。
